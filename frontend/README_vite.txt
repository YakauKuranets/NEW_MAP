frontend/ — скелет Vite-проекта для Map v12
==========================================

Это подготовка к Этапу 5.1: "Сборка (Vite) + структура фронта".

Что здесь есть сейчас:
----------------------
* package.json  — минимальные dev-зависимости и скрипты:
    - npm install
    - npm run dev
    - npm run build
* vite.config.js — сборка из src/ в ../static/dist
* index.html — отдельная Vite-страница для песочницы (НЕ основная страница Flask).
* src/main.js — входная точка Vite, сейчас только лог и задел под будущее.
* src/style.css — пробный файл стилей для Vite-песочницы.

Как использовать (черновой режим):
----------------------------------
1) Перейти в каталог frontend:
     cd frontend
2) Установить зависимости:
     npm install
3) Запустить dev-сервер:
     npm run dev

На этом этапе Vite НИКАК не вмешивается в боевую страницу карты.
Flask по-прежнему отдаёт templates/index.html и тянет static/js/*.js,
как раньше. Это сделано специально, чтобы сначала спокойно поднять
инфраструктуру сборки, а уже затем пошагово переносить модули в Vite.

Следующие шаги по Этапу 5.1:
----------------------------
* Вынести часть модулей в ES-модули внутри frontend/src.
* Настроить сборку в static/dist и начать подключать один бандл
  вместо "гирлянды" <script src="..."> в index.html.
* Добавить кэш-бастинг и, при необходимости, manifest.json для шаблона Flask.


Зеркало текущих JS-модулей
--------------------------
В каталоге frontend/src/modules/ находятся копии существующих фронтовых
модулей из static/js/:

  main.js
    chat.js
    analytics.js
    requests.js
    filters.js
    offline.js
    map_core.js
    addresses.js
    export.js
    sidebar.js
    search.js
    ui.js

Это сделано как подготовка к постепенной миграции на ES-модули под Vite:
* на следующем шаге можно будет по одному модулю адаптировать под import/export;
* пока эти файлы НИКУДА не подключены и не влияют на боевую страницу карты;
* Flask по-прежнему использует static/js/*.js напрямую.

Таким образом, можно безопасно экспериментировать с Vite и переписыванием
модулей, не рискуя боевым интерфейсом.
